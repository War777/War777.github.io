Kellogg's Billateral KPIs

Java MySql Ubuntu Server Apache

En Kellogg Company, en conjunto con el departamento de Customer Service del área logística, se identificaron causas raíces que afectaban el Fill Rate, siendo las entregas tardías (‘Late Deliveries’, Cada entrega tardía afectaba en la facturación por $500K promedio) una razón de impacto.

Se trazó un plan de acción para trabajar con los proveedores de transporte para llevar métricas compartidas con el departamento de logística (KPIs Bilaterales) que permitieran medir el desempeño con la finalidad de entregar incentivos a los transportistas con mejoría y reducción en sus entregas tardías.

Se logró reducir el número de entregas tardías en 30% así mismo la evaluación permitió desarrollar un programa de premiación semestral el cual alentaba a los proveedores a seguir mejorando sus procesos.

Se desarrolló una aplicación de escritorio que permitía el flujo de información entre ambos lados, compañía y proveedor la cual eliminaba el proceso de cálculos manuales por parte de los ejecutivos del departamento de ‘customer service’ permitiéndoles centrarse en la toma de decisiones.

Vi-Pro Server

Php Laravel Ubuntu Server DigitalOcean MariaDB Apache Github jQuery

Se logró mejorar la estrategia de comunicación y acción y reducir el tiempo de solución por parte del departamento de servicios municipales.

Se generó una plataforma de consulta hacia la ciudadanía que daba transparencia en la recaudación de recursos y status de procesos.

Se liberó al personal de trabajo repetitivo y manual para que pudieran centrarse en estrategias de mejora continua.

Generamos nuestra navaja Suiza para los procesos internos.

Kellogg's - Cost To Serve

Php Laravel Ubuntu Server Mysql Apache Github jQuery

En Kellogg Company, surge la necesidad de crear un modelo de negocio que busca maximizar la utilidad de principio a fin de la cadena de valor a través de la segmentación de costos bajo una perspectiva de mercado.

Por ello se decidió crear una herramienta que ayude identificar la forma en que los costos generados a todo nivel posible (Marca, Producto, Cliente, Cuenta, etc.), para poder determinar decisiones que permitieran tomar acciones en la mejora de los procesos.

Se logró mejorar el gasto general del departamento de Logística siendo el modelo un ejemplo de los resultados que se logran al tener un trabajo coordinado entre las áreas y jefaturas.
